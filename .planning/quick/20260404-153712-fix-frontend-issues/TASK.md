# 快速任务：修复前端两个交互问题

## 问题描述

### 问题1：摄像头停止后仍运行
**现象**：点击“停止摄像头”按钮后，前端停止显示视频流，但后端摄像头仍在运行（可以听到摄像头声音或观察到指示灯）。

**根本原因**：`_stream_camera`函数中的`while True:`循环没有停止机制。前端只是清除了`streamUrl`，停止请求新的帧，但后端的`cv2.VideoCapture`仍在读取摄像头数据。

**影响**：摄像头资源未释放，可能影响性能和电池寿命。

### 问题2：文件上传后重复弹窗  
**现象**：上传图片或视频文件并开始检测后，文件选择对话框会再次弹出。

**根本原因**：`FileInput.vue`组件的`reset()`方法只清空了input的value，但：
1. `selectedFile.value`可能仍然保留文件引用
2. 点击检测按钮可能触发了某些事件
3. 文件拖拽事件处理可能有问题

**影响**：用户体验不佳，每次检测都需要关闭额外的弹窗。

## 解决方案

### 问题1解决方案：添加摄像头停止机制
- 添加一个全局标志或状态来停止`_stream_camera`循环
- 修改`_stream_camera`函数，使其可以响应停止信号
- 或者使用不同的方法管理摄像头生命周期

### 问题2解决方案：修复文件输入重置逻辑
- 改进`FileInput.vue`的`reset()`方法，完全清除状态
- 确保`selectedFile.value`在检测后重置为`null`
- 检查事件冒泡和默认行为
- 可能需要延迟重置或更改事件处理顺序

## 验证方法

### 摄像头停止验证：
1. 启动摄像头，观察预览正常
2. 点击停止按钮，预览区域显示“预览区域”
3. 检查后端Flask日志，确认摄像头读取循环已停止
4. 物理摄像头指示灯应熄灭（如果支持）

### 文件上传验证：
1. 选择图片/视频文件
2. 点击检测按钮
3. 确认文件选择对话框不再自动弹出
4. 可以正常选择新文件进行下一次检测
